# Research: Preamble Clarification Phase

## Decision: Preamble questions reuse the public clarification mechanism

**Rationale**: The app already has a visible user answer panel, resume context, and public question events. Reusing the same mechanism keeps the experience consistent and avoids introducing a second answer flow.

**Alternatives considered**: A separate preamble chat surface was rejected for now because it would duplicate validation, display, and resume behavior before the interaction model is proven.

## Decision: Pause before turn 1 when preamble questions exist

**Rationale**: If the council asks for clarification before debate, it should not also offer advice from the unclarified prompt. This makes the pause meaningful.

**Alternatives considered**: Letting some mentors deliberate while others ask questions was rejected for this slice because it muddies the ceremony and transcript.
